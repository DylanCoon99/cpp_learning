# Step 13: Templates

## Concepts

Templates let you write code that works with any type. They're C++'s version of generics — similar in purpose to Go's generics, but much more powerful (and more complex).

### 1. Function Templates

```cpp
// Without templates: write one per type
int max_int(int a, int b) { return (a > b) ? a : b; }
double max_double(double a, double b) { return (a > b) ? a : b; }

// With templates: write once, works for any type
template <typename T>
T max_val(T a, T b) {
    return (a > b) ? a : b;
}

max_val(3, 7);          // T = int (deduced)
max_val(3.14, 2.71);    // T = double (deduced)
max_val<int>(3, 7);     // T = int (explicit)
```

`template <typename T>` says "T is a placeholder for any type." The compiler generates a **separate function** for each type used — this is called **instantiation**.

`typename` and `class` are interchangeable here:
```cpp
template <typename T>  // same as:
template <class T>     // both mean "T is a type"
```

### 2. Multiple Template Parameters

```cpp
template <typename T, typename U>
auto add(T a, U b) {
    return a + b;   // return type deduced with auto
}

add(3, 4.5);    // T=int, U=double, returns double
```

### 3. Class Templates

```cpp
template <typename T>
class Stack {
public:
    void push(const T& value) {
        data_.push_back(value);
    }

    T pop() {
        T top = data_.back();
        data_.pop_back();
        return top;
    }

    bool empty() const { return data_.empty(); }
    int size() const { return data_.size(); }

private:
    std::vector<T> data_;
};

Stack<int> int_stack;
int_stack.push(42);

Stack<std::string> str_stack;
str_stack.push("hello");

Stack<Stack<int>> nested;  // stack of stacks — works!
```

Each instantiation (`Stack<int>`, `Stack<std::string>`) is a completely separate class generated by the compiler.

### 4. Non-Type Template Parameters

Template parameters don't have to be types — they can be values:

```cpp
template <typename T, int Size>
class FixedArray {
public:
    T& operator[](int i) { return data_[i]; }
    constexpr int size() const { return Size; }
private:
    T data_[Size];   // size is known at compile time
};

FixedArray<int, 10> arr;     // array of 10 ints
FixedArray<double, 3> vec;   // array of 3 doubles

// This is essentially what std::array<T, N> does
```

### 5. Template Specialization

You can provide a custom implementation for a specific type:

```cpp
// General template
template <typename T>
std::string to_string(const T& value) {
    return std::to_string(value);  // works for int, double, etc.
}

// Specialization for std::string (std::to_string doesn't work on strings)
template <>
std::string to_string<std::string>(const std::string& value) {
    return value;  // already a string
}

// Specialization for bool
template <>
std::string to_string<bool>(const bool& value) {
    return value ? "true" : "false";
}
```

### 6. Partial Specialization (Classes Only)

```cpp
// General template
template <typename T>
class Wrapper {
    T value_;
public:
    Wrapper(T v) : value_(v) {}
    void print() const { std::cout << value_ << "\n"; }
};

// Partial specialization for pointers
template <typename T>
class Wrapper<T*> {
    T* ptr_;
public:
    Wrapper(T* p) : ptr_(p) {}
    void print() const {
        if (ptr_) std::cout << *ptr_ << "\n";
        else std::cout << "nullptr\n";
    }
};

int x = 42;
Wrapper<int> w1(42);     // uses general template
Wrapper<int*> w2(&x);    // uses pointer specialization
```

### 7. Where to Put Template Code

**Templates must be defined in the header file** (or in the same file they're used). Unlike regular functions, you can't put the declaration in a `.h` and definition in a `.cpp`. This is because the compiler needs to see the full template definition to instantiate it for each type.

```cpp
// my_stack.h — everything goes here
template <typename T>
class Stack {
public:
    void push(const T& value) { data_.push_back(value); }
    T pop() { /* ... */ }
private:
    std::vector<T> data_;
};

// my_stack.cpp — NO. Templates can't be split like this.
```

For now (single-file exercises), this doesn't matter. It'll become important in Phase 4 when we cover project structure.

### 8. Template Argument Deduction (TAD)

The compiler can usually figure out `T` from the arguments:

```cpp
template <typename T>
T square(T x) { return x * x; }

square(5);      // T = int (deduced from argument)
square(3.14);   // T = double (deduced)
square<long>(5); // T = long (explicit — overrides deduction)
```

For class templates, C++17 added **CTAD** (Class Template Argument Deduction):

```cpp
std::vector v = {1, 2, 3};    // deduces vector<int>
std::pair p = {42, "hello"};  // deduces pair<int, const char*>
```

### 9. `auto` with Templates

```cpp
template <typename T, typename U>
auto multiply(T a, U b) -> decltype(a * b) {
    return a * b;
}

// -> decltype(a * b) is a trailing return type.
// It says: "the return type is whatever a * b produces"
// In C++14+, you can just use auto without the trailing type:

template <typename T, typename U>
auto multiply(T a, U b) {
    return a * b;
}
```

### 10. Variadic Templates (Brief Introduction)

Templates that take any number of arguments:

```cpp
// Base case
template <typename T>
T sum(T value) {
    return value;
}

// Recursive case
template <typename T, typename... Args>
T sum(T first, Args... rest) {
    return first + sum(rest...);
}

sum(1, 2, 3, 4, 5);          // 15
sum(1.5, 2.5, 3.0);          // 7.0
```

`Args...` is a **parameter pack** — zero or more types. `rest...` expands the pack. This is advanced — just know it exists.

### 11. `constexpr` with Templates

Templates + `constexpr` = compile-time computation:

```cpp
template <int N>
constexpr int factorial() {
    if constexpr (N <= 1) return 1;
    else return N * factorial<N - 1>();
}

constexpr int f = factorial<5>();  // computed at compile time: 120
```

`if constexpr` (C++17) evaluates the condition at compile time and discards the other branch entirely — unlike regular `if`.
